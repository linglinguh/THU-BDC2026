# 通知公告

### 关于提交代码须由模型预测产出结果的规则提醒

**各位参赛团队：
在近期提交的代码复现与审查过程中，我们发现个别团队的最终提交结果并非由模型对输入数据预测产生，而是将固定的股票代码、权重等结果直接写入代码后原样输出。为避免更多团队对规则理解出现偏差，现就比赛规则中的相关要求作重点提醒，请各团队务必对照自查。**

#### 一、规则要求：结果必须由模型训练和预测产生

根据比赛规则，参赛作品的主要贡献应为机器学习方法，包括模型的训练和预测。提交的代码必须包含完整、真实的模型训练与预测流程，最终提交的 result.csv 必须是由模型基于输入数据预测产生的结果。

#### 二、以下情形属于违规

1. 将固定的股票代码、持仓权重等最终结果直接硬编码在代码、模型文件或配置文件中，绕过模型训练和预测流程直接输出；
2. 提交的代码虽然可以运行、输出结果可以复现，但输出结果与输入数据之间不存在实际的预测依赖关系，即结果不随输入数据变化，实质上仍是预置结果的搬运。
   请注意：「代码可以运行、结果可以复现」不等于合规。复现要求的是模型训练与预测过程的可复现，而不是预置结果的重复输出。

#### 三、复现阶段将进行代码审查

在最终验证阶段，赛事方将在运行复现的基础上进行代码审查。凡经审查认定最终结果不是由模型预测产生的团队，将按比赛规则取消其成绩及获奖资格。
本提醒面向全体参赛团队，不针对任何特定团队。如自查发现存在上述违规情形，请及时整改。

### 关于最终验证阶段数据目录挂载规则的重要提醒

各位参赛团队：
在近期提交的代码审核过程中，我们发现部分团队因未正确打包所需数据，导致复现失败。为避免更多团队在最终验证阶段出现同样问题，现就代码规范中的数据目录挂载规则进行重点提醒，请各位务必仔细阅读并按要求调整。

#### 一、最终验证将严格按代码规范挂载 data 目录

根据代码规范，项目结构中 data 目录为外部挂载目录：
|--data（储存数据，在 docker-compose.yml 中进行挂载）
在最终验证阶段，赛事方将严格按照代码规范执行：镜像加载后，data 目录会被挂载为赛事方提供的数据。该挂载操作会覆盖选手镜像中原有的 data 目录内容，请各位不要将自己依赖的数据存放在 data 目录内。

#### 二、赛事方提供的数据内容

最终验证时，data 目录中将挂载以下内容：

- stock_data：截至提交最后一天往前 3 年的数据，供训练和验证使用；
- 不提供 train和 test 数据。

#### 三、两种数据使用方式及注意事项

##### 1.方式1：使用赛事方提供的 stock_data

如果选手希望使用赛事方挂载的 stock_data，需要在自己的代码中完成数据划分（例如划分训练集、验证集等），并基于划分后的数据进行模型训练和验证。

##### 2.方式2：使用选手自行准备的数据

如果选手不希望使用赛事方提供的数据，而是使用自己准备的数据，请务必将这些数据放置到不会被挂载的目录中。
根据代码规范，建议存放位置为：
|--model（储存训练模型和其它数据，docker 内）
即：将自定义数据打包到 model 目录下，或代码目录等不会被 docker-compose.yml 挂载的目录中。请勿放在 data、output、temp 这三个挂载目录内，否则将在最终验证时被覆盖。

#### 四、请尽快自查并调整

**请各位团队在最终提交前，对照代码规范自查：
1.data 目录中是否存放了不可被覆盖的数据？
2.若使用官方 stock_data，代码中是否已完成数据划分与训练流程？
3.若使用自有数据，是否已放置在 model 等非挂载目录中？**

### 2026大数据挑战赛docker提交方式变更通知

各参赛团队、各位选手：
近日，在线上赛A阶段首次模型代码提交过程中，组委会接到反馈，部分队伍上传docker所使用的网盘链接出现失效情况。为了给大家提供更稳定、便捷的提交体验，经组委会商议，决定对docker提交流程进行优化，具体调整如下：

### 1.实行“先结果、后代码”的分步提交

- 不再要求队伍同时提交result.csv与docker文件。
- 后续所有队伍统一**先行提交result.csv结果文件**。平台将根据得分进行排名，**排名靠前的队伍进入docker提交环节**（线上赛A阶段每次选取前 20名，线上赛B阶段选取各奖项名额上限的 120%）。

### 2.开放平台专属上传入口

- 需要提交docker的入围队伍不再使用网盘提交，竞赛平台届时将开放专属上传入口，选手直接在系统内上传docker即可。
- 上传入口开放时间为线上赛每次排行榜公布日的次日。
- A阶段首次成绩排行榜发布及docker上传
  （1）成绩排行榜发布时间：2026年5月5日
  （2）前20队伍docker上传时间：2026年5月6日10:30-23:59
  （3）前20队伍docker上传入口：https://www.heywhale.com/org/r9xi8/competition/area/69c0dfa34f302f8f0122e1bb/submit
  （4）其余阶段成绩排行榜发布及docker上传时间另行通知。

### 3.重申“完全一致”的代码复现要求

- 代码审核环节将严格执行可复现性审查。选手提交的模型代码运行生成的结果必须与此前在平台提交的result.csv结果完全一致。对于复现结果和提交结果不一致或代码不可复现的队伍，组委会将剔除其成绩并有权取消其获奖资格。



### 【特别说明】

本次调整仅针对docker提交方式，大赛其余赛程安排、数据规范及评审标准均保持不变。感谢各位选手的理解与配合！因网盘突发状况给大家带来的不便，组委会深表歉意。预祝各位选手在后续的比赛中进展顺利，取得优异成绩！
